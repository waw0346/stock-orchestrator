## 🎯 1. 문제 현상 및 핵심 원인 진단

장중 테스트 과정에서 **일부 프로세스가 무한 대기 상태에 빠지고**, 코드 실행 시 **사용자 승인 대화상자가 과도하게 호출되는 현상**이 식별되었습니다. 이에 대해 시스템 자가진단을 실시한 결과, 다음과 같은 아키텍처적 취약점이 규명되었습니다.

### 🔴 원인 A: 하위 프로세스 분리 기동에 따른 승인 오버헤드 (Subprocess Spawning)
*   **현상:** 하나의 파이프라인([run_us_close_korea_pipeline.ps1](file:///c:/Users/kjw03/Desktop/stock%20orchestrator/scripts/run_us_close_korea_pipeline.ps1))을 실행하는데 사용자 허가 팝업이 7번 이상 연달아 발생합니다.
*   **원인:** 파이프라인 스크립트 내부에서 다른 스크립트를 실행할 때, **`powershell -NoProfile -ExecutionPolicy Bypass -File ...`** 명령을 사용하여 **새로운 파워쉘 독립 프로세스를 계속 생성(Spawn)**하고 있습니다.
*   **결과:** 샌드박스 보안 모델은 새로운 프로세스가 뜰 때마다 독립적인 명령어로 판단하여 사용자에게 승인을 요구합니다. 7단계의 파이프라인이 진행되는 동안 7번의 프로세스 승인 팝업이 뜨는 물리적 비효율성이 발생합니다.

### 🔴 원인 B: 대화형 콘솔 대기(Interactive Blocking)에 의한 무한 루프
*   **현상:** 백그라운드 태스크가 완료되지 않고 1시간 이상 `RUNNING` 상태로 멈춰 있는 현상이 발생했습니다 (예: 작업 340).
*   **원인:** [run_weekly_flow_momentum_update.ps1](file:///c:/Users/kjw03/Desktop/stock%20orchestrator/scripts/run_weekly_flow_momentum_update.ps1)과 같은 스크립트들이 실행 도중 사용자에게 파일 생성 또는 아카이브 여부를 묻는 **대화형 입력 프롬프트(Read-Host, Y/N)**를 노출하고 있습니다.
*   **결과:** 표준 입력(stdin)이 연결되지 않은 백그라운드 모드에서 스크립트가 실행될 경우, 사용자가 답변을 입력할 방법이 없으므로 프로세스는 답변을 기다리며 **영원히 대기(Hanging)** 상태에 빠지게 됩니다.

### 🔴 원인 C: 프로세스 타임아웃의 부재 (Lack of Timeout Guard)
*   **현상:** API 호출이나 스크립트 구동 에러 시 프로세스가 즉시 종료되지 않고 계속해서 자원을 점유한 채 대기합니다.
*   **원인:** Python 및 PowerShell 호출 시 프로세스 타임아웃 제어 장치가 부재하여, 세션 연결이 끊기거나 에러가 났을 때 강제 종료되지 못하고 좀비 프로세스로 남아 있게 됩니다.

## 🛠️ 2. 구조적 개선 및 해결 방안 (Refactoring Plan)

시스템의 메타인지 자가 발전 목적에 맞추어, **수동 개입을 최소화하고 실행 안정성을 확보**하기 위한 3대 개선책을 제시합니다.

### 1) 닷소싱(Dot-Sourcing) 및 호출 연산자(&)로의 전면 전환
*   **해결안:** 하위 스크립트를 독립 프로세스로 기동하는 `powershell -File script.ps1` 구문을 폐기하고, 현재 파워쉘 세션 내부에서 실행하는 **닷소싱(`.`) 또는 호출 연산자(`&`)**로 대체합니다.
*   **개선 효과:**
    1.  **사용자 승인 1회 축소:** 하위 프로세스가 새로 뜨지 않으므로, 최상위 파이프라인 실행 시 **단 1회의 승인**만으로 하위의 모든 연산이 진행됩니다.
    2.  **메모리 및 환경 변수 자연 공유:** 토큰값이나 세션 정보를 공용 메모리나 현재 세션 변수에서 그대로 활용할 수 있어, 무거운 `.env.local` 파싱 과정이 최소화됩니다.
    3.  **좀비 프로세스 원천 예방:** 부모 세션이 종료되면 내부에 바인딩된 모든 하위 작업이 동일 스레드 상에서 즉각 소멸하므로 백그라운드에 프로세스가 갇히는 현상이 사라집니다.

```powershell
# AS-IS (하위 프로세스 생성 - 매번 승인 요구, 좀비 프로세스 유발)
powershell -NoProfile -ExecutionPolicy Bypass -File (Join-Path $scriptDir "refresh_kiwoom_token.ps1")

# TO-BE (현재 프로세스 내에서 실행 - 승인 요구 없음, 안정적인 생명주기)
. (Join-Path $scriptDir "refresh_kiwoom_token.ps1")
```

### 2) 헤드리스/비대화형 실행(Non-Interactive Mode) 강제화
*   **해결안:** 파이프라인에서 호출되는 모든 스크립트 매개변수에 `-NoPrompt` 스위치를 필수 전달하고, 스크립트 내부에서는 백그라운드 감지 시 입력을 생략하고 디폴트값으로 처리하도록 예외 로직을 구성합니다.
*   **개선 효과:** 수동 입력을 대기하는 프롬프트 차단으로 백그라운드 무한 대기 현상이 100% 방지됩니다.

### 3) 샌드박스 내부 스케줄러 및 타임아웃 가드 설치
*   **해결안:** Python의 `subprocess` 호출 시 `timeout` 파라미터를 강제 적용하고, PowerShell 실행 시 `Wait-Process`에 `-Timeout` 파라미터를 설정하여 지정된 시간(예: 60초) 내에 완료되지 않는 프로세스는 운영체제 레벨에서 자동으로 강제 종료(`Kill`)되도록 방어벽을 설치합니다.
